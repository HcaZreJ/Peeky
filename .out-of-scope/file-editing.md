# 文件编辑

本项目不做编辑功能——产品定位是只读查看器。

## Why this is out of scope

Peeky 的核心价值是"`peek <path>` 弹出即可读"（time-to-window ≤450ms）的最快只读预览。编辑能力会拖入保存、undo 栈、编码写回、文件权限、外部并发修改冲突检测一整条链路，与零状态渲染架构（纯函数渲染管线 + 单一 UI 控制器）和冷启预算直接冲突。

"看完顺手改两行"的需求由 **⌘E 用系统默认编辑器打开当前文件（VS Code/Cursor 识别时带 `path:line` 行号定位）** 承接——见 plan `2026-07-03-peek-native.md` 单元 R4b。文本的**选中与复制**不属于编辑，属于查看器的应有能力，已在 R4a/R4b 覆盖。
